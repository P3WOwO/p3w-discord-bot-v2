# P3W Discord Bot

## Что это
Discord-бот с:
- командой времени в войсе
- топом по войсу
- жизнью бота
- админскими командами `msg`, `purge`, `jtm`
- AI-ответами через Gemini
- хранением состояния в Supabase

## Что нужно создать
1. Проект в Supabase
2. Таблицу из `supabase_bot_state.sql`
3. Web Service в Render

## Переменные окружения
Обязательные:
- `TOKEN`
- `CLIENT_ID`
- `GUILD_ID`
- `SUPABASE_URL`
- `SUPABASE_SERVICE_ROLE_KEY`

Для AI:
- `GEMINI_API_KEY`
- `GEMINI_MODEL` (необязательно, по умолчанию `gemini-2.5-flash-lite`)
- `GEMINI_MODEL_FALLBACK` (необязательно, по умолчанию `gemini-2.5-flash`)
- `GEMINI_IMAGE_MODEL` (необязательно, по умолчанию `gemini-2.5-flash-image`)
- `GEMINI_IMAGE_MODEL_FALLBACK` (необязательно, по умолчанию `gemini-3.1-flash-image`)
- `BOT_BASE_PROMPT` (необязательно, базовый стиль общения)

Дополнительно:
- `SUPABASE_TABLE` (необязательно, по умолчанию `bot_state`)
- `SUPABASE_ROW_ID` (необязательно, по умолчанию `main`)

## Память ИИ
Система памяти теперь хранит:
- краткую общую сводку,
- память по пользователю и по каналу,
- отдельный профиль бота с базовым промптом общения, тоном и стилем,
- авто-удаление дублей и слабых/устаревших заметок.

Бот сам решает, что сохранить, а что выкинуть, чтобы контекст не разрастался.


## Память ИИ v2
Теперь память работает в несколько слоёв:
- быстрый heuristic-pass ловит устойчивые факты даже без Gemini;
- Gemini пишет структурированный JSON для долговременной памяти;
- память разделяет предпочтения, ограничения, проекты и факты;
- спорные записи уходят в pending review;
- есть базовый промпт общения, который можно посмотреть и менять через `persona`;
- text/image модели умеют переключаться на fallback при ограничениях или ошибках лимита;


## Новые команды
- `/image` — генерация картинки по описанию
- `/persona` — показать или поменять базовый промпт общения
- `!img`, `!image`, `!imagine` — быстрый запуск генерации картинки
- `!persona` — показать или поменять базовый промпт общения
